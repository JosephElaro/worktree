# 🧰 worktree - Simple version control for Windows

[⬇️ Download worktree](https://github.com/JosephElaro/worktree/releases)

## 🚀 What is worktree?

worktree is a version control app for managing files, branches, and access rules in one place. It is built for people who want a clear way to track changes, work with branches, and keep project access under control.

This app is made for Windows users who want to download a release and run it with little setup. It fits personal projects, team folders, and larger shared workspaces.

## 📥 Download worktree

Visit this page to download: https://github.com/JosephElaro/worktree/releases

On that page, look for the latest release and download the Windows file that matches your computer. If you see more than one file, choose the one for Windows with the .exe or .msi file type.

## 🖥️ System needs

Before you install worktree, check these basic needs:

- Windows 10 or Windows 11
- A modern 64-bit PC
- At least 200 MB of free disk space
- A stable internet connection for the first download
- Permission to run apps on your computer

For best results, close other large apps before you install.

## 🧭 How to install on Windows

1. Open the download page: https://github.com/JosephElaro/worktree/releases
2. Find the newest release at the top of the page
3. Open the Assets section
4. Download the Windows file
5. If Windows asks, choose Keep or Run
6. If you downloaded an .exe file, double-click it
7. If you downloaded an .msi file, double-click it and follow the setup screen
8. If Windows shows a security prompt, choose Run anyway only if the file came from the release page above
9. Wait for the install to finish
10. Open worktree from the Start menu or desktop shortcut

## 📂 How worktree fits your workflow

worktree helps you organize work in a way that feels simple:

- Use a tree view to keep files in order
- Create branches to separate changes
- Control who can view or edit shared areas
- Keep track of version history
- Store project settings in a TOML file when needed
- Manage code changes without a lot of manual steps

This makes it useful for solo work and for teams that need clear access rules.

## 🔐 Access controls

worktree includes access control features for shared use. These help you decide who can open, edit, or manage parts of a project.

Common use cases:

- Give one user edit rights
- Let a group view a project but not change it
- Protect sensitive folders
- Split access by branch
- Keep admin actions limited to trusted users

If you manage shared work, start with a small set of users and add more as needed.

## 🌳 Branches and tree view

Branches let you test changes without affecting the main version. The tree view gives you a clear way to see how your files are arranged.

Use these features to:

- Keep new work separate
- Review changes before you merge them
- Organize large projects
- Find files faster
- Reduce mistakes when many people work on the same project

If you know how to use folders, you already understand the basic idea of the tree view.
